# Stratum Bitcoin Solo Miner ⛏️

קליינט כרייה (Miner) שנכתב ב-Node.js, מתחבר ל-Mining Pool דרך פרוטוקול Stratum V1 ומבצע כריית סולו (Solo Mining) על המעבד (CPU) תוך שימוש בריבוי תהליכונים (Multi-threading).

> **⚠️ אזהרה/הבהרה:** כריית ביטקוין ב-2026 דורשת חומרת ASIC ייעודית. מעבד רגיל (CPU) מסוגל לבצע רק מאות אלפי חישובים בשנייה (KH/s), בעוד שרשת הביטקוין פועלת בקצב של Exahashes. הסיכוי הסטטיסטי למצוא בלוק בעזרת מחשב אישי הוא אפסי. הפרויקט הזה הוא **פרויקט לימודי** להבנת פרוטוקול Stratum והקריפטוגרפיה שמאחורי ביטקוין.

---

## 🛠️ מה צריך כדי להתחיל?

כיוון שהגדרנו מראש את `public-pool.io` (פול ייעודי לכריית סולו שאינו דורש הרשמה או סיסמה), הדבר היחיד שאתה צריך כדי להתחיל הוא **כתובת ארנק ביטקוין**. הפול מזהה אותך לפי כתובת הארנק שאליה הוא ישלח את ה-3.125 BTC במקרה הנדיר שתמצא בלוק.

### שלב 1: יצירת ארנק ביטקוין
אם אין לך ארנק, תצטרך לפתוח אחד. הנה שתי אפשרויות קלות ובטוחות:
*   **Electrum (מומלץ למחשב):** תוכנת ארנק קוד-פתוח, ותיקה ובטוחה. חפש "Electrum Bitcoin Wallet" בגוגל, הורד, כתוב בצד את 12 המילים של הגיבוי, ולחץ על "Receive" כדי לקבל כתובת.
*   **Exodus / Trust Wallet:** ארנקים עם ממשק מודרני (לטלפון או למחשב). 

העתק את הכתובת שלך. היא לרוב תתחיל באותיות `bc1q...` או בספרות `1` או `3`.

### שלב 2: עדכון ההגדרות בקוד
בתיקיית הפרויקט קיים קובץ בשם `.env`. פתח אותו עם כל עורך טקסט ושנה את השורה של הכתובת:
```env
POOL_HOST=public-pool.io
POOL_PORT=3333
BTC_ADDRESS=הדבק_כאן_את_הכתובת_שלך
WORKER_NAME=my-awesome-miner
HTTP_PORT=3224
```

## 🚀 איך מפעילים ומכבים? (מנגנון המיינר השקט)

כדי שהמיינר לא יפריע לך ביום-יום (בזמן לימודי JS, צפייה בסרטים וכו'), הגדרנו מנגנון אינטראקטיבי שלא מעמיס על המחשב שלך:

### 1. הפעלה אוטומטית בהדלקת המחשב (אינטראקטיבית)
בכל פעם שתדליק את המחשב, תקפוץ חלונית קטנה שתשאל אותך: **"האם ברצונך להפעיל את כריית הביטקוין כעת?"**:
*   **לחצת "לא כרגע":** המיינר לא יידלק ולא יצרוך שום כוח מהמעבד.
*   **לחצת "כן, תפעיל":** ייפתח סרגל גרירה שבו תוכל לבחור כמה ליבות מעבד (מתוך ה-12 הקיימות במחשב שלך) להקצות לטובת הכרייה. שאר הליבות יישארו פנויות לחלוטין לעבודה השוטפת שלך.

### 2. עצירה מהירה משולחן העבודה (Desktop Shortcut)
על שולחן העבודה שלך מחכה כפתור קיצור דרך אדום בשם **`עצור כרייה 🛑`**:
*   דאבל קליק עליו יעצור מיד את המיינר, ישחרר את כל הליבות של המעבד ויציג לך הודעה שהכרייה הופסקה.

## 💻 פקודות ידניות לטרמינל (לשליטה מלאה)

אם אתה מעדיף להשתמש בטרמינל כדי להפעיל או לעצור באופן ידני:

### הפעלת המיינר ידנית (עם החלונית לבחירת ליבות):
פקודה זו תפתח את החלונית לבחירת הליבות ותריץ את המיינר ברקע:
```bash
./startup-prompt.sh
```

### הפעלת המיינר ידנית (הרצה ישירה בטרמינל עם הפלט לקריאה):
אם אתה רוצה לראות את הקוד רץ ומדווח בטרמינל עצמו (ללא ריצה ברקע):
```bash
node stratum-miner.mjs
```

### עצירת המיינר ידנית (הריגת התהליך):
פקודה זו תכבה את המיינר ותשחרר את כל ליבות המעבד מיד:
```bash
./stop-miner.sh
```
או ישירות דרך פקודת המערכת:
```bash
pkill -f stratum-miner.mjs
```

## 🌐 לוח בקרה אינטרנטי (מעקב אחרי הכסף)

בזמן שהמיינר רץ, לוח הבקרה זמין בדפדפן שלך בכתובת:
👉 **[http://localhost:3224](http://localhost:3224)**

שם תוכל לראות בזמן אמת:
*   **זמן ריצה** וקצב גיבוב כולל (Hashrate) משולב של כל הליבות.
*   **מניות (Shares) שנשלחו והתקבלו** בהצלחה ב-Pool (זה מוכיח שהחישוב עובד תקין).
*   **ביטקוין שהורווח:** בכריית סולו זה יעמוד על `0.00000000 ₿` ויקפוץ בבת אחת ל-`3.125 ₿` רק אם תמצא בלוק שלם בכוחות עצמך.

## 📊 שמירת נתונים וסטטיסטיקות (stats.json)

הנתונים הסטטיסטיים של הכרייה (זמן ריצה, מניות שנמצאו, Hashrate ועוד) נשמרים באופן מקומי בלבד בקובץ `stats.json`.
* הקובץ אינו נשמר או נשלח למאגר ה-Git המשותף (מוגדר ב-`.gitignore`), כך שכל מחשב שומר על הנתונים הייחודיים שלו ללא סכנת קונפליקטים.
* הנתונים נשמרים וממשיכים לצבור ערכים גם בין הפעלות חוזרות של המיינר.
* **איפוס מונים:** כדי לאפס את כל הנתונים חזרה לאפס, ניתן פשוט למחוק את הקובץ `stats.json` (הוא ייווצר מחדש מאופס בהפעלה הבאה) או לרוקן את ערכיו לפי המבנה המופיע בקובץ הדוגמה `stats.example.json`.

## 🔒 אבטחה וחיבור מבוזר (Security)

כדי למנוע גישה לא מורשית ושליטה חיצונית במיינר או שליחת מניות מזויפות, המערכת מוגנת על ידי מנגנון אבטחה:
1. **טוקן שיתוף סודי (DASHBOARD_TOKEN)**: בהפעלה הראשונה נוצר אוטומטית מפתח אבטחה אקראי שנשמר בקובץ ה-`.env`. כל בקשה לביצוע פעולות או קבלת נתונים (ממכשירי טלפון או מהדפדפן) חייבת לכלול את הטוקן בכותרת `X-Auth-Token`.
2. **האזנה בטוחה (`BIND_HOST`)**: כברירת מחדל השרת מאזין ל-`127.0.0.1` (localhost בלבד). אם ברצונך לחבר מכשירי טלפון ברשת המקומית, עליך לעדכן ב-`.env` את ההגדרה `BIND_HOST=0.0.0.0`.
3. **חיבור מחוץ לרשת הביתית**: אם ברצונך לחבר וורקרים שנמצאים מחוץ לבית (למשל, טלפון ברשת סלולרית או בעיר אחרת), **אל תבצע הפניית פורטים (Port Forwarding)** פתוחה לאינטרנט. מומלץ להשתמש בפתרון VPN מאובטח וחינמי כמו **Tailscale** כדי ליצור רשת פרטית מאובטחת בין המחשב לטלפון. בחיבור כזה תוכל להריץ את ה-phone-worker עם כתובת ה-IP של ה-PC שנוצרה בתוך רשת ה-Tailscale באופן בטוח לחלוטין.
